# 🚑 VANET-Based Emergency Vehicle Alert System

> **BTech Final Year Project · VANET · SUMO · Python TraCI**

A Python simulation of **Vehicle-to-Vehicle (V2V) communication** where an ambulance broadcasts its presence over a VANET network and automatically alerts nearby vehicles — implemented on top of [Eclipse SUMO](https://sumo.dlr.de/) via the TraCI API on a realistic 5 × 5 signalised grid network.

---

## ✨ Key Features

| Feature | Details |
|---|---|
| 🚑 Emergency vehicle priority | `vClass="emergency"` + TraCI signal preemption — ambulance never stops at red lights |
| 📡 VANET broadcast model | Beacon every 0.5 s, 100 m alert range, ±70° forward FOV cone |
| 🚗 Continuous traffic | `<flow>` elements generate 30–40 cars throughout the entire 300 s simulation |
| ⏱ Time-to-Collision | Closing-speed TTC estimated each step; lane-change advice issued when TTC < 10 s |
| 🎨 Live GUI coloring | Red ambulance · Yellow alerted · Blue normal · Green just-cleared |
| 🕹 Speed control | `--delay` flag lets you slow the simulation for demos or presentations |
| 📊 Metrics report | Total alerted, average alert distance, TTC log printed at the end |

## 🗺 Network Overview

```
j04 ── j14 ── j24 ── j34 ── j44     ↑ Y
 |      |      |      |      |       |
j03 ── j13 ── j23 ── j33 ── j43     |  800 m
 |      |      |      |      |       |
j02 ── j12 ── j22 ── j32 ── j42   ← Ambulance corridor (row 2)
 |      |      |      |      |
j01 ── j11 ── j21 ── j31 ── j41
 |      |      |      |      |
j00 ── j10 ── j20 ── j30 ── j40   → X  (800 m)
```

- Every node is a **traffic-light** controlled junction
- Every edge has **2 lanes**, speed limit 50 km/h (13.89 m/s)
- Ambulance travels **row 2** (j02 → j12 → j22 → j32 → j42)

## 🏗 Architecture

```
main.py  ──►  SUMO-GUI / SUMO  (via TraCI)
    │
    │  every 0.5 s step
    ▼
vanet_logic.py  ─  VANETSystem.process_step()
    │
    ├── 1. Read all vehicle positions from TraCI
    ├── 2. Identify ambulance
    ├── 3. For each other vehicle:
    │       a. Euclidean distance to ambulance
    │       b. Is vehicle in ±70° forward cone? (is_vehicle_ahead)
    │       c. Distance ≤ 100 m AND ahead → ALERT
    │       d. Compute Time-to-Collision (TTC)
    │       e. TTC < 10 s AND dist < 60 m → changeLane / slowDown via TraCI
    ├── 4. Apply GUI colours via traci.vehicle.setColor()
    └── 5. Return StepResult with metrics

utils.py  ─  euclidean_distance · is_vehicle_ahead · time_to_collision
             log_alert · log_ambulance · log_metrics
```

## ⚙️ Prerequisites

### 1. Install Eclipse SUMO

**Windows** — Download the MSI installer from:
<https://sumo.dlr.de/docs/Downloads.php>

**Ubuntu / Debian:**
```bash
sudo add-apt-repository ppa:sumo/stable
sudo apt-get update && sudo apt-get install sumo sumo-tools sumo-doc
```

**macOS (Homebrew):**
```bash
brew install sumo
```

### 2. Set `SUMO_HOME`

**Windows (PowerShell — run once, then restart terminal):**
```powershell
[Environment]::SetEnvironmentVariable(
  "SUMO_HOME", "C:\Program Files (x86)\Eclipse\Sumo", "User"
)
```

**Linux / macOS:**
```bash
export SUMO_HOME=/usr/share/sumo          # adjust to your install path
echo 'export SUMO_HOME=/usr/share/sumo' >> ~/.bashrc
```

### 3. Install Python packages

```bash
pip install traci sumolib
```

> Python **3.8 or newer** is required.

## 🚀 Running the Simulation

### GUI mode (recommended for demos)
```bash
cd files
python main.py
```
SUMO-GUI opens automatically and the simulation starts. The ambulance spawns at **t = 5 s** in red; alerted vehicles turn yellow.

### Headless mode (fast, no window)
```bash
python main.py --no-gui
```

### Slow it down for a presentation
```bash
# 200 ms per step  (~120 s wall-clock for 600 steps)
python main.py --delay 200

# 500 ms per step  (~300 s wall-clock — every event clearly visible)
python main.py --delay 500
```

### All CLI options

| Flag | Default | Description |
|---|---|---|
| `--no-gui` | off | Run SUMO headless (no graphical window) |
| `--steps` | `600` | Total simulation steps (1 step = 0.5 s sim time) |
| `--delay MS` | `100` (GUI) / `0` (headless) | Milliseconds to pause between each step |

> **Tip:** With `--delay 100` (the GUI default) a 600-step run takes ~60 s of real time — comfortable for live demos.

## 🔬 VANET Alert Logic

```
Alert fires when ALL conditions are true:
  ✅  Euclidean distance(ambulance, vehicle) ≤ 100 m
  ✅  Vehicle lies within ±70° forward cone of ambulance heading
```

### GUI Colour Scheme

| Colour | Meaning |
|---|---|
| 🔴 Red | Ambulance |
| 🟡 Yellow | Vehicle currently inside alert zone |
| 🟢 Green | Vehicle just exited alert zone |
| 🔵 Blue | Normal vehicle (not alerted) |

### Traffic-Light Preemption (Double Layer)

1. **`vClass="emergency"`** in `routes.rou.xml` — SUMO automatically grants green-light priority to the ambulance at every junction.
2. **TraCI override** in `main.py` — when the ambulance is within 80 m of a junction, `traci.trafficlight.setRedYellowGreenState()` forces an all-green phase as an additional safety net.

## 📊 Metrics Explained

| Metric | Description |
|---|---|
| Total vehicles alerted | Unique vehicles that received at least one VANET alert |
| Average alert distance | Mean distance (m) at which vehicles were first alerted |
| Simulation duration | Total simulated time in seconds |

| Wall-clock time | Real elapsed time including any `--delay` pauses |

## 🛠 Troubleshooting

| Problem | Fix |
|---|---|
| `sumo-gui: command not found` | Add SUMO `bin/` folder to PATH; verify `SUMO_HOME` is set |
| `ImportError: No module named traci` | `pip install traci sumolib` |
| Ambulance stops at red lights | Ensure `vClass="emergency"` is in the ambulance vType in `routes.rou.xml` |
| No vehicles appear | Check `routes.rou.xml` is sorted by departure time (flows before the ambulance vehicle entry) |
| Unicode/emoji error on Windows terminal | Set `PYTHONUTF8=1` or run `python main.py` from Windows Terminal (supports UTF-8) |
| Simulation too fast to watch | Use `--delay 200` or higher |
